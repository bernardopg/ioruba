## Design Context

### Users
Primary users are makers, Linux desktop power users, streamers, and audio tinkerers who want a tactile way to control system audio with cheap Arduino hardware. They use the app while desk-bound, often with multiple audio targets active at once, and need immediate confidence that the hardware, serial link, and audio backend are all alive.

### Brand Personality
Industrial, confident, legible. The interface should feel like a precision instrument rather than a generic dashboard or gamer control panel. Emotional goal: calm control with obvious operational feedback.

### Aesthetic Direction
Instrument-panel / studio-lab direction with warm copper and cool teal accents, clear visual hierarchy, visible telemetry, and tactile circular knob motifs. Default away from neon gamer aesthetics, glassmorphism, and generic SaaS card grids.

### Design Principles
1. Connection state must be impossible to miss.
2. Hardware readings should feel physical and alive, not abstract.
3. Configuration should stay powerful but remain recoverable through defaults and visible JSON.
4. Diagnostics should be first-class, not hidden behind secondary tooling.
5. The UI should look like purpose-built hardware software, not a repurposed admin panel.
